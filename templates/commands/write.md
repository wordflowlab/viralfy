# /write - Newsletter 创作系统

## 概述

Newsletter 创作系统支持三种创作模式,并提供增量生成和断点续传功能。这是 Viralfy 的核心功能。

## 第一步: 执行脚本检查状态

执行以下命令来检查创作状态:

```bash
bash scripts/bash/write.sh
```

脚本会返回当前的创作状态和进度信息。

## 第二步: 解析返回结果

```json
{
  "status": "success",
  "action": "create" | "resume",
  "data": {
    "newsletter_id": "issue-001",
    "progress": {
      "total_sections": 6,
      "completed_sections": [1, 2],
      "current_section": 3,
      "word_count": 1200
    }
  }
}
```

## 最佳实践

1. **使用研究材料**: 始终基于深度研究创作,避免内容空洞
2. **增量保存**: 每完成一个章节就保存,避免丢失
3. **多轮迭代**: 初稿完成后,至少进行一轮审核和优化
4. **保持结构**: 遵循 Newsletter 的标准结构(钩子-观点-实践-总结)
5. **注意字数**: 建议总字数 3000-5000 字,保持深度和可读性

## 创作技巧

### 钩子写作 (Hook)
- **个人经历**: 从真实经历出发,建立共鸣
- **痛点挖掘**: 挖掘读者的核心痛点
- **承诺明确**: 清楚告诉读者能获得什么

### 核心观点 (Core Insights)
- **反直觉**: 挑战常识,提供新视角
- **有论据**: 数据、案例、权威引用
- **可理解**: 复杂概念用简单类比

### 实践指南 (Actionable Steps)
- **具体明确**: 每个步骤都可执行
- **循序渐进**: 从易到难排列
- **工具推荐**: 提供具体工具和资源

### 总结 CTA (Call to Action)
- **回顾要点**: 3-5 个核心要点
- **明确行动**: 告诉读者下一步做什么
- **降低门槛**: 让第一步足够简单

## 故障排除

### 问题 1: 创作中断

**解决方案**: 下次运行 `viralfy write` 会自动检测并提供恢复选项

### 问题 2: 找不到研究材料

**解决方案**: 先运行 `viralfy research` 创建知识库

### 问题 3: AI 生成质量不佳

**解决方案**:
- 提供更详细的创作要求
- 加载更多研究材料
- 使用"从零创作"模式,手动控制质量

## 相关命令

- `viralfy research` - 深度研究,构建知识库
- `viralfy distribute` - 将 Newsletter 分发到多平台
- `viralfy review` - 内容质量评估
- `viralfy status` - 查看创作进度

## 快捷命令

在创作过程中可以使用的快捷命令:

- `:save` - 保存当前章节
- `:preview` - 预览当前内容
- `:skip` - 跳过当前章节
- `:back` - 返回上一章节
- `:help` - 显示帮助信息
- `:quit` - 退出创作(进度会保存)
