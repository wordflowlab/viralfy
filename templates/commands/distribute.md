## 第二步: 初始化分发流程

获取用户选择后 (假设选择了 Newsletter 1):

```bash
# AI 操作: 初始化分发
bash scripts/bash/distribute.sh issue-001
```

### 解析返回结果

脚本返回:
- `action`: "init_distribution"
- `newsletter_id`: Newsletter ID
- `newsletter_file`: Newsletter 文件路径
- `distribution_dir`: 分发目录路径
- `platforms`: 配置的目标平台列表 (从 config.json 读取)

### 读取 Newsletter 内容

```bash
# AI 操作: 读取 Newsletter
cat newsletters/issue-001/newsletter.md
```

**AI 必须完整理解 Newsletter 的**:
- 标题和主题
- 核心观点 (3-5 个)
- 关键论据和案例
- 整体结构和逻辑
- 目标受众和价值主张

## 💡 分发技巧

### 1. 平台差异化

不是简单复制粘贴,而是根据平台特点改写:

| 平台 | 特点 | 内容策略 |
|------|------|----------|
| Twitter | 简短、互动性强 | 金句 + 观点 + Thread |
| 小红书 | 攻略、视觉化 | 实用 + 分段 + Emoji |
| 公众号 | 深度、专业 | 完整文章 + 详细论证 |
| 视频 | 视觉、节奏 | 分镜 + 钩子 + B-roll |
| 知乎 | 理性、数据 | 分析 + 引用 + 专业性 |

### 2. 生成多个变体

每个平台生成 2-3 个不同风格的版本:
- 数据导向 vs. 故事导向
- 教学风格 vs. 经验分享
- 正式语气 vs. 个人化语气

测试不同版本的表现,找到最优风格。

### 3. 保留核心价值

无论如何改写,确保:
- 核心观点一致
- 价值主张清晰
- 逻辑结构完整
- 个人风格保留

### 4. 适配平台规则

- Twitter: 280 字符限制
- 小红书: 1000 字限制
- 视频: 时长控制 (5-10 分钟)
- 公众号: 排版美观
- 知乎: 专业术语和引用

### 5. 添加平台特定元素

- Twitter: @提及、#话题标签
- 小红书: Emoji、分段、标签
- 公众号: 互动引导、关注提示
- 视频: 钩子、B-roll、字幕
- 知乎: 数据图表、引用来源
